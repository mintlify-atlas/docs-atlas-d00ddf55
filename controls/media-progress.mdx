---
title: "Media and Progress Controls in WinUI: Playback and Feedback"
description: "Use ProgressBar, ProgressRing, AnimatedIcon, AnimatedVisualPlayer, and WebView2 to show loading states, animate icons, and host web content in WinUI apps."
---

WinUI media and progress controls cover the visual feedback layer of your application — from a simple determinate bar to a full web content host. These controls are optimized for the Windows Composition layer and integrate with WinUI's theming system.

## ProgressBar

`ProgressBar` is a horizontal bar that communicates progress toward a goal. It extends `RangeBase`, giving it `Value`, `Minimum`, and `Maximum` properties. Setting `IsIndeterminate` to `true` switches to an animated shimmer that indicates activity without a defined end point. `ShowError` and `ShowPaused` apply distinct visual states without changing the `Value`.

<Tabs>
  <Tab title="Determinate">
```xml
<muxc:ProgressBar
    x:Name="UploadBar"
    Value="65"
    Minimum="0"
    Maximum="100"
    Width="300" />
```
  </Tab>
  <Tab title="Indeterminate">
```xml
<muxc:ProgressBar
    IsIndeterminate="True"
    Width="300" />
```
  </Tab>
  <Tab title="Error state">
```xml
<muxc:ProgressBar
    Value="45"
    ShowError="True"
    Width="300" />
```
  </Tab>
  <Tab title="Paused state">
```xml
<muxc:ProgressBar
    Value="30"
    ShowPaused="True"
    Width="300" />
```
  </Tab>
</Tabs>

### Key properties

| Property | Type | Default | Description |
|---|---|---|---|
| `Value` | `Double` | — | Current progress value (inherited from `RangeBase`). |
| `Minimum` | `Double` | `0` | Lower bound of the range. |
| `Maximum` | `Double` | `100` | Upper bound of the range. |
| `IsIndeterminate` | `Boolean` | `false` | Switch to the animated indeterminate state. |
| `ShowError` | `Boolean` | `false` | Apply an error visual state (typically red). |
| `ShowPaused` | `Boolean` | `false` | Apply a paused visual state (typically yellow). |

<Note>
`ShowError` and `ShowPaused` are visual-only states. They do not stop or reset the underlying `Value`.
</Note>

## ProgressRing

`ProgressRing` is a circular spinner control. Like `ProgressBar`, it supports both indeterminate (spinning) and determinate (arc that fills based on a value) modes. It is `IsActive` by default.

<Tabs>
  <Tab title="Indeterminate">
```xml
<muxc:ProgressRing
    IsActive="True"
    IsIndeterminate="True"
    Width="40"
    Height="40" />
```
  </Tab>
  <Tab title="Determinate">
```xml
<muxc:ProgressRing
    IsActive="True"
    IsIndeterminate="False"
    Value="60"
    Minimum="0"
    Maximum="100"
    Width="60"
    Height="60" />
```
  </Tab>
</Tabs>

### Key properties

| Property | Type | Default | Description |
|---|---|---|---|
| `IsActive` | `Boolean` | `true` | Whether the ring is animating. Set to `false` to pause. |
| `IsIndeterminate` | `Boolean` | `true` | `true` for a spinning animation; `false` for a value-driven arc. |
| `Value` | `Double` | `0` | Current value (used when `IsIndeterminate` is `false`). |
| `Minimum` | `Double` | `0` | Lower bound for the value range. |
| `Maximum` | `Double` | `100` | Upper bound for the value range. |

## AnimatedIcon

`AnimatedIcon` is an `IconElement` subclass that plays a Lottie-compatible animated visual source. The animation is driven by a named state string set via the `AnimatedIcon.State` attached property. A `FallbackIconSource` is displayed when the animated source cannot be loaded.

<Tabs>
  <Tab title="Basic">
```xml
<!-- Assuming AnimatedBackVisualSource is generated by LottieGen -->
<muxc:AnimatedIcon>
    <muxc:AnimatedIcon.Source>
        <animatedvisuals:AnimatedBackVisualSource />
    </muxc:AnimatedIcon.Source>
    <muxc:AnimatedIcon.FallbackIconSource>
        <muxc:SymbolIconSource Symbol="Back" />
    </muxc:AnimatedIcon.FallbackIconSource>
</muxc:AnimatedIcon>
```
  </Tab>
  <Tab title="State-driven">
```xml
<!-- Set state to drive the animation; states are defined in the Lottie file markers -->
<Button
    muxc:AnimatedIcon.State="Normal"
    PointerEntered="Button_PointerEntered"
    PointerExited="Button_PointerExited">
    <muxc:AnimatedIcon x:Name="BackIcon">
        <animatedvisuals:AnimatedBackVisualSource />
    </muxc:AnimatedIcon>
</Button>
```

```csharp
private void Button_PointerEntered(object sender, PointerRoutedEventArgs e)
{
    AnimatedIcon.SetState(BackIcon, "PointerOver");
}

private void Button_PointerExited(object sender, PointerRoutedEventArgs e)
{
    AnimatedIcon.SetState(BackIcon, "Normal");
}
```
  </Tab>
</Tabs>

### Key properties

| Property | Type | Description |
|---|---|---|
| `Source` | `IAnimatedVisualSource2` | The animated visual source (generated by LottieGen or similar). |
| `FallbackIconSource` | `IconSource` | Fallback shown if the source fails to load. |
| `MirroredWhenRightToLeft` | `Boolean` | Flip the icon horizontally in right-to-left layouts. |

### Attached property

| Property | Description |
|---|---|
| `AnimatedIcon.State` | A string that selects the animation segment to play, defined by markers in the Lottie file. Set on the `AnimatedIcon` element or an ancestor. |

<Tip>
The state string values (e.g., `"Normal"`, `"PointerOver"`, `"Pressed"`) must match the marker names in the Lottie JSON file used to generate the visual source.
</Tip>

## AnimatedVisualPlayer

`AnimatedVisualPlayer` is a standalone playback host for `IAnimatedVisualSource` content. Unlike `AnimatedIcon`, it is not constrained to icon sizes and provides direct playback control methods: `PlayAsync`, `Pause`, `Resume`, `Stop`, and `SetProgress`.

<Tabs>
  <Tab title="Auto-play looping">
```xml
<!-- AnimatedHeartVisualSource generated by LottieGen -->
<muxc:AnimatedVisualPlayer
    AutoPlay="True"
    PlaybackRate="1"
    Stretch="Uniform"
    Width="200"
    Height="200">
    <animatedvisuals:AnimatedHeartVisualSource />
</muxc:AnimatedVisualPlayer>
```
  </Tab>
  <Tab title="Manual playback">
```xml
<StackPanel>
    <muxc:AnimatedVisualPlayer
        x:Name="Player"
        AutoPlay="False"
        Stretch="Uniform"
        Width="200"
        Height="200">
        <animatedvisuals:AnimatedCheckMarkVisualSource />
    </muxc:AnimatedVisualPlayer>

    <Button Content="Play" Click="PlayButton_Click" />
    <Button Content="Pause" Click="PauseButton_Click" />
</StackPanel>
```

```csharp
private async void PlayButton_Click(object sender, RoutedEventArgs e)
{
    await Player.PlayAsync(fromProgress: 0, toProgress: 1, looped: false);
}

private void PauseButton_Click(object sender, RoutedEventArgs e)
{
    Player.Pause();
}
```
  </Tab>
</Tabs>

### Key properties

| Property | Type | Default | Description |
|---|---|---|---|
| `Source` | `IAnimatedVisualSource` | — | The animated visual to play (content property). |
| `AutoPlay` | `Boolean` | `true` | Start playing immediately when the source is loaded. |
| `PlaybackRate` | `Double` | `1` | Playback speed multiplier. Negative values play in reverse. |
| `Stretch` | `Stretch` | `Uniform` | How the visual fills the player bounds. |
| `IsPlaying` | `Boolean` | — | Read-only. `true` while an animation is playing. |
| `IsAnimatedVisualLoaded` | `Boolean` | — | Read-only. `true` when the source has been successfully loaded. |
| `Duration` | `TimeSpan` | — | Read-only. Total duration of the loaded animation. |
| `FallbackContent` | `DataTemplate` | — | Rendered when no source is set or loading fails. |
| `AnimationOptimization` | `PlayerAnimationOptimization` | `Latency` | `Latency` (low-latency start) or `Resources` (lower memory). |

### Methods

| Method | Description |
|---|---|
| `PlayAsync(from, to, looped)` | Play from `from` progress (0–1) to `to` progress, optionally looping. Returns an awaitable. |
| `Pause()` | Pause the current animation at the current frame. |
| `Resume()` | Resume a paused animation. |
| `Stop()` | Stop playback and return to the start frame. |
| `SetProgress(progress)` | Seek to a specific progress position (0–1) without playing. |

## WebView2

`WebView2` embeds a Chromium-based Microsoft Edge WebView2 control inside a WinUI window. It renders full web content and provides a two-way bridge between the WinUI application and JavaScript running inside the web view.

<Tabs>
  <Tab title="Navigate to URL">
```xml
<muxc:WebView2
    x:Name="WebView"
    Source="https://example.com" />
```
  </Tab>
  <Tab title="Navigate in code">
```csharp
// After the control is loaded:
WebView.Source = new Uri("https://example.com");

// Or navigate with NavigateToString for local HTML:
WebView.NavigateToString("<html><body><h1>Hello</h1></body></html>");
```
  </Tab>
  <Tab title="Call JavaScript">
```csharp
// Execute script after the page has loaded
private async void WebView_NavigationCompleted(
    WebView2 sender,
    CoreWebView2NavigationCompletedEventArgs e)
{
    string result = await sender.ExecuteScriptAsync("document.title");
    PageTitleText.Text = result.Trim('"');
}
```
  </Tab>
</Tabs>

<Warning>
`WebView2` requires the WebView2 Runtime to be installed on the user's machine. It is included by default in Windows 11 and available as a separate download for Windows 10.
</Warning>

### Key properties

| Property | Type | Description |
|---|---|---|
| `Source` | `Uri` | The URL to navigate to. Setting this triggers navigation. |
| `CoreWebView2` | `CoreWebView2` | Access to the underlying CoreWebView2 for advanced configuration (available after `EnsureCoreWebView2Async` completes). |
| `CanGoBack` | `Boolean` | Whether the web view can navigate backward. |
| `CanGoForward` | `Boolean` | Whether the web view can navigate forward. |
| `DefaultBackgroundColor` | `Color` | Background color shown before page content loads. |

### Common methods

| Method | Description |
|---|---|
| `EnsureCoreWebView2Async()` | Initialize the underlying WebView2 environment. Await before accessing `CoreWebView2`. |
| `ExecuteScriptAsync(script)` | Run a JavaScript expression and return its result as a JSON string. |
| `GoBack()` / `GoForward()` | Navigate the back/forward history stack. |
| `Reload()` | Reload the current page. |
| `NavigateToString(html)` | Load an HTML string directly into the view. |

### Key events

| Event | Description |
|---|---|
| `NavigationStarting` | Fires before navigation begins; allows cancellation. |
| `NavigationCompleted` | Fires when navigation succeeds or fails. |
| `CoreWebView2Initialized` | Fires when the `CoreWebView2` object is ready to use. |
