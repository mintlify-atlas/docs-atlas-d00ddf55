WinUI 3 is the native UI framework for Windows desktop apps, shipping as part of the Windows App SDK. This guide walks you through everything you need to go from a fresh machine to a running WinUI 3 app.

## System requirements

Before you begin, make sure your machine meets these requirements:

| Requirement | Minimum version |
|---|---|
| Windows | Windows 10, version 1809 (build 17763) or later |
| Visual Studio | Visual Studio 2022 (any edition) |
| Workload | Windows App SDK (installed via Visual Studio Installer) |

<Note>
Windows 11 is recommended for the best development experience, but Windows 10 1809 and later are fully supported for both development and deployment.
</Note>

## Create your first WinUI 3 app

<Steps>
  <Step title="Install the Windows App SDK workload">
    Open the **Visual Studio Installer** from your Start menu. Select **Modify** next to your Visual Studio 2022 installation.

    In the **Workloads** tab, find and check **Windows application development**. This installs the Windows App SDK components, WinUI 3 project templates, and related tools.

    Click **Modify** to apply the changes. Wait for the installation to complete, then close the installer.

    <Tip>
      If Visual Studio 2022 is not yet installed, download it from [visualstudio.microsoft.com](https://visualstudio.microsoft.com/) and select the **Windows application development** workload during initial setup.
    </Tip>
  </Step>

  <Step title="Create a new WinUI 3 project">
    Open Visual Studio 2022 and select **Create a new project** from the start window.

    In the project template search box, type `WinUI`. Select the **Blank App, Packaged (WinUI 3 in Desktop)** template, then click **Next**.

    <Info>
      The "Packaged" template includes an application package manifest and is the recommended starting point for distributing apps through the Microsoft Store or via MSIX.
    </Info>

    Configure your project:
    - **Project name**: `MyWinUIApp` (or any name you prefer)
    - **Location**: choose a folder on your machine
    - **Solution name**: leave as the default or customize

    Click **Create**. Visual Studio generates a solution with a `MainWindow.xaml` file and a code-behind `MainWindow.xaml.cs`.
  </Step>

  <Step title="Explore the generated project structure">
    After the project is created, Solution Explorer shows the key files:

    - **`App.xaml` / `App.xaml.cs`** — application entry point and lifecycle
    - **`MainWindow.xaml`** — the main window XAML layout
    - **`MainWindow.xaml.cs`** — code-behind for the main window
    - **`Package.appxmanifest`** — MSIX package identity and capabilities

    The default `MainWindow.xaml` already contains a `StackPanel` with a `Button` and a `TextBlock`. You will modify these in the next step.
  </Step>

  <Step title="Add controls in XAML and wire up a click handler">
    Open `MainWindow.xaml`. Replace the contents of the `StackPanel` element with the following markup:

    <CodeGroup>

    ```xml MainWindow.xaml
    <StackPanel
        Orientation="Vertical"
        HorizontalAlignment="Center"
        VerticalAlignment="Center"
        Spacing="12">

        <TextBlock
            x:Name="myTextBlock"
            Text="Hello, WinUI 3!"
            FontSize="24"
            HorizontalAlignment="Center" />

        <Button
            x:Name="myButton"
            Content="Click me"
            HorizontalAlignment="Center"
            Click="Button_Click" />

    </StackPanel>
    ```

    ```csharp MainWindow.xaml.cs
    private void Button_Click(object sender, RoutedEventArgs e)
    {
        myButton.Content = "Clicked!";
        myTextBlock.Text = "You clicked the button.";
    }
    ```

    </CodeGroup>

    The `x:Name` attribute assigns a name you can reference in code-behind. The `Click` attribute wires the button to the `Button_Click` method in `MainWindow.xaml.cs`.
  </Step>

  <Step title="Run the app">
    Press **F5** (or select **Debug > Start Debugging**) to build and launch the app.

    Visual Studio compiles the project, packages it, and deploys it to your local machine. A window appears showing your `TextBlock` and `Button`. Click the button to see both controls update.

    <Check>
      If the window appears and the button responds to clicks, your WinUI 3 environment is set up correctly.
    </Check>

    To run without the debugger attached, press **Ctrl+F5** or select **Debug > Start Without Debugging**.
  </Step>
</Steps>
